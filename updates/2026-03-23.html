SDK SOV: Impact on IAA Campaigns

Key Findings

1. SDK SOV is the clearest early predictor of IAA success

  • Near-target campaigns: ~80% SDK SOV on their tracking bundle.
  • Underperformers: ~60% SDK SOV.
  • SDK SOV <50% is almost never seen among near-target campaigns — a red-flag zone.
  • Practical OS guidance:
    • iOS: aim for ≥55% SDK SOV
    • Android: aim for ≥65% SDK SOV

2. SDK daily spend on the bundle matters

  • From the 6th decile (~>$1.5k/day SDK spend), >50% of campaigns are near-target.
  • Very low spend (~$30/day) can still work but is too noisy; $1.5k/day+ is a practical benchmark.

3. Rewarded via SDK helps, but isn’t a gate

  • Performing campaigns have ~45% vs ~35% of spend on rewarded via SDK (statistically significant).
  • Successful campaigns exist with 0% rewarded — treat as a nice-to-have uplift, not a requirement.

4. Closed-loop/subsidy metrics are promising but post-launch only

  • Subsidy, % boosted impressions correlate positively with ROAS, but depend on live closed-loop users.
  • Better as post-launch health signals and for future user-level analyses.

How Sales Should Use This

  • Check the tracking bundle’s SDK SOV:
    • Green: iOS ≥55%, Android ≥65% → much higher odds of near-target ROAS.
    • Red flag: SDK SOV <50% → historically very few successes; position as high-risk / experimental.
  • Check SDK daily demand spend:
    • Prefer ~$1.5k/day+ (≥6th decile) → more than half of campaigns hit ≥90% of target.
  • Use rewarded share as an optimization lever, not a gate:
    • Increasing rewarded share via SDK is directionally good, but don’t block IAA activation because of low rewarded share.
